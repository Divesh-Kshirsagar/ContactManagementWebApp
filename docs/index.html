üìö API Documentation

Base URL: https://contact-manager-api-production-387e.up.railway.app/api/v1

Endpoints

GET /healthcheck - Check API status
GET /contacts - Get all contacts with pagination
Query params: page, limit, search, category
GET /contacts/:id - Get single contact
POST /contacts - Create new contact
Body: { name, email, phone, category, message? }
PUT /contacts/:id - Update contact
Body: { name?, email?, phone?, category?, message? }
DELETE /contacts/:id - Delete contact
POST /contacts/bulk-delete - Delete multiple contacts
Body: { ids: string[] }

Example Request

POST /api/v1/contacts
Content-Type: application/json

{

  "email": "john@example.com",

üì± Validation Rules

Field Type Required Validation

email string Yes Valid email format
phone string Yes 10-15 digits, international format
category enum Yes Work, Family, Friends, Other
message string No Max 500 characters

Phone Number Formats

Accepted formats:

üèóÔ∏è Architecture

MVC Pattern (Backend)

  • Models - Mongoose schemas and database models
  • Views - JSON responses handled by controllers
  • Controllers - Handle HTTP requests/responses
  • Services - Business logic layer
  • Validators - Zod validation schemas
  • Middlewares - Error handling, validation

Client-Side Filtering Strategy

  • ‚â§1000 contacts: Fetches all contacts once, filters/searches client-side (instant results)
  • >1000 contacts: Server-side pagination, search, and filtering

Security Features

  • Helmet.js - Secure HTTP headers
  • CORS - Configured for specific origins
  • Input Validation - Zod validation on both client and server
  • MongoDB Injection Protection - Mongoose sanitization
  • Environment Variables - Sensitive data protection

cd ContactManagementWebApp

# Install dependencies

# Setup environment variables (see .env.example files)

npm run dev

Development Scripts

# Install all dependencies
npm run install:all

# Development
npm run dev              # Run both client and server
npm run dev:client       # Run client only
npm run dev:server       # Run server only

# Build
npm run build            # Build both
npm run build:client     # Build client only
npm run build:server     # Build server only

# Production
npm start                # Start production server
